# TEYEON Roadmap

실제 개발 우선순위를 관리하는 문서입니다.

## Phase 1 안정화

- KDK Realtime 동기화 유지 및 회귀 방지
- localhost 로그인 redirect 수정 상태 유지
- 로그인/권한 구조를 `profiles.role` 단일 기준으로 전환
- 신규 로그인자는 `GUEST`로 생성
- CEO가 관리자 화면에서 권한을 수동 부여하는 흐름 정리
- 기존 이메일/env/members 기반 자동 권한 상승 제거
- 배포 전 진단 로그 최소화

## Phase 2 공개/체험 구조

- MEMBER/GUEST도 KDK 생성 과정을 볼 수 있는 Preview Mode 설계
- Preview Mode는 실제 DB 저장 없이 대진 생성 흐름을 체험하는 구조로 검토
- 게스트가 앱의 가치를 이해할 수 있는 공개 접근 범위 정의
- 권한별 메뉴 노출과 접근 차단 정책 정리
- 카카오 프로필 사진은 표시용으로만 유지

## 별도 Phase 회원 공개 준비

- [ ] 회원용 사용 매뉴얼 제작
- [ ] 운영진용 사용 매뉴얼 제작
- [ ] Figma 기반 화면 흐름 문서 검토
- [ ] 카카오톡 공유용 간단 매뉴얼 이미지 제작
- [ ] PDF 매뉴얼 제작 검토
- [ ] 게스트/정회원/운영진 권한별 사용 방법 정리

## Phase 3 디자인 개선

- Luxury Sports Club Dashboard 방향으로 전체 UI 톤 정리
- 홈, KDK, 관리자, 멤버 프로필의 시각적 위계 통일
- 운영 화면은 현장 사용성을 우선해 빠르고 명확하게 개선
- 카드, 버튼, 상태 표시의 디자인 규칙 정리
- 모바일 현장 사용성 점검

## Phase 4 확장

- `club_id` 기반 멀티 클럽 구조 확장
- 장기적으로 `club_memberships` 모델 검토
- 클럽별 멤버, 권한, 세션, 아카이브 분리
- 다른 클럽 도입 가능성을 고려한 설정 구조 정리
- 유료화 가능성 검토: 클럽 운영 도구, 대회 관리, 기록/랭킹 아카이브
