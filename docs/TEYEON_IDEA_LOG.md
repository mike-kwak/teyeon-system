# TEYEON Idea Log

앱 개발 중 떠오르는 아이디어를 보관하는 문서입니다. 확정된 작업은 `TEYEON_ROADMAP.md`로 옮겨 우선순위를 관리합니다.

## 디자인

- Luxury Sports Club Dashboard 방향으로 전체 톤 개선
- 경기 운영 화면은 현장 사용성을 우선하고, 정보 밀도와 가독성을 높이기
- 관리자/운영자 화면은 조용하고 단단한 컨트롤 센터 느낌으로 정리
- 멤버와 게스트가 보는 화면은 프리미엄 클럽 앱처럼 명확하고 가볍게 구성

## KDK

- KDK Realtime 동기화 성공 상태 유지
- 경기 투입, 점수 저장, 다른 브라우저 자동 반영 흐름 안정화
- MEMBER/GUEST도 KDK 생성 과정을 볼 수 있는 Preview Mode 장기 과제로 검토
- Preview Mode에서는 실제 저장 없이 대진 생성 흐름과 결과를 체험하게 하기
- 운영자는 실시간 경기 상태와 랭킹을 빠르게 확인할 수 있게 개선

## 로그인/권한

- 카카오 로그인은 유지
- 로그인 후 현재 경로로 돌아오도록 redirect 유지
- 앱 권한 판단 기준은 `profiles.role`로 단일화
- 신규 로그인자는 기본 `GUEST`
- CEO가 관리자 화면에서 `MEMBER`/`ADMIN` 권한을 수동 부여
- 카카오 닉네임, 이메일, 전화번호 자동 매칭은 권한 판단에 사용하지 않음
- 카카오 프로필 사진은 표시용으로만 사용

## 게스트/홍보

- GUEST도 앱의 분위기와 KDK 흐름을 볼 수 있는 공개/체험 구조 검토
- 가입 전 사용자가 클럽 운영 방식과 경기 시스템을 자연스럽게 이해하도록 구성
- 홍보용 화면은 과장된 랜딩보다 실제 앱 경험을 보여주는 방향 선호
- Preview Mode를 게스트 전환 장치로 활용 가능

## 회원 공개 / 매뉴얼

### 아이디어

- 앱이 어느 정도 안정화되면 테연 클럽 회원들에게 공개하기 전에 사용 매뉴얼을 만든다.
- 회원들이 처음 접속했을 때 로그인, 메뉴 이동, KDK 대진 확인, 라이브 경기 확인, 랭킹 확인 방법을 쉽게 이해할 수 있어야 한다.
- 운영진용 매뉴얼과 일반 회원/게스트용 매뉴얼을 분리하는 것을 검토한다.
- 매뉴얼 제작 방식으로 Figma를 검토한다.
- Figma로 화면 흐름, 클릭 위치, 권한별 사용 방법을 시각적으로 정리하면 회원들에게 공유하기 좋다.
- 최종적으로 PDF, 이미지 카드, 인스타/카카오톡 공유용 요약본으로도 변환할 수 있게 한다.

### 목적

- 회원 공개 시 혼란 줄이기
- 운영진 교육 시간 줄이기
- 앱의 완성도와 신뢰감 강화
- 나중에 다른 클럽 확장/유료화 시 온보딩 자료로 재사용

## 장기 확장

- `club_id` 기반 멀티 클럽 확장 고려
- 장기적으로 `club_memberships(user_id, club_id, role, member_id)` 구조 검토
- 다른 클럽이 자체 멤버, 권한, KDK 세션을 운영할 수 있는 구조 검토
- 유료화 가능성: 클럽별 운영 도구, 대회 관리, 기록 보관, 프리미엄 디자인

## 나중에 할 일

- Realtime 진단 로그 최종 정리
- `members.role`과 `profiles.role` 의미 완전 분리
- 관리자 화면에서 `profiles.role` 직접 관리 구조 추가
- KDK Preview Mode 설계
- 홈 메뉴 권한 표시 방식 정리
- 운영/로컬 환경변수 분리 문서화
